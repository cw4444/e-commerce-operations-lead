# Contributing

Thanks for helping improve Signal Forge.

## Good first changes

- Real data connectors for marketplaces, reviews, email, or Slack
- Better scoring rules for listing health and account health
- New report formats for weekly or monthly ops reviews
- Improved onboarding flows for new brands

## Workflow

1. Open an issue or start from an existing one.
2. Keep changes small and focused.
3. Run `npm run build` before opening a pull request.
4. Explain the user impact clearly in the PR description.

## Principles

- Prefer practical automation over decorative AI
- Keep the dashboard readable at a glance
- Make follow-up actions explicit
- Design new integrations so they can be swapped out later
